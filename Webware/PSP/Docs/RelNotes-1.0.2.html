Python Server Pages 1.0.2 Release Notes

Python Server Pages version 1.0.2 released on 06/07/09

New Features

  • The file extensions of the files that shall be parsed as PSP can now be configured using the setting ExtensionsForPSP (suggested by Tobias Franz). By default (as before), only files with the extension .psp are regarded as PSP files.
  • You can now use an arbitrary encoding for your PSP files if you specify the encoding in the PSPFileEncoding setting.

Improvements and Refinements

  • Empty PSP tags are now silently ignored.

Bugfixes

  • When using the psp-handler of mod_webkit, request methods like servletPath() and others returned values which were different from what you get when using the ordinary webkit-handler. This could cause various problems, for instance because the default cookie path is set to the servlet path (reported by Fionn Behrens). To avoid such problems, requests made via the psp-handler now will look much more similar to a request made via the ordinary webkit-handler.
