TaskKit 1.0 Release Notes

TaskKit version 1.0 released on 11/29/08

Improvements and Refinements

  • The Scheduler now accepts an exceptionHandler argument. If passed, this exception handler is invoked whenever an unhandled exception occurs within a Task. Previously there was no general mechanism for handling exceptions within Tasks.
  • If an exception occurs within a periodic Task, the Task will be started again at the next scheduled time. Previously when an exception occurred, the Task was not able to be restarted.

Bugfixes

  • The isOnDemand() method of the TaskHandler did not work correctly.
  • The scheduler could not unregister running tasks (reported as bug #849732 by Brett Funderburg).
